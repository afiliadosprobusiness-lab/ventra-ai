# Ventra Design System

## Brand intent

Ventra debe sentirse como un Revenue OS moderno, claro y comercial. La base actual prioriza limpieza visual y velocidad de lectura por encima de densidad dramatica.

## Core tokens

### Color roles

- Primary: verde/teal Ventra
- Background: neutro claro para la app
- Card: blanco con borde sutil
- Accent: variaciones suaves del primary
- Success / warning / info: estados funcionales, no decorativos

### Typography

- Display: `Space Grotesk`
- Body/UI: `Manrope`

### Radius

- Base radius: `0.625rem`
- Cards y botones comparten redondeado medio y consistente

## Surface recipes

### Primary card

Uso:
- KPIs
- tablas resumidas
- bloques de insight
- pricing

Caracteristicas:
- fondo blanco
- borde fino
- sombra corta y sobria

### CTA surfaces

Uso:
- botones primarios
- chips destacados
- logos de marca

Caracteristicas:
- gradiente verde Ventra
- contraste alto

## Shell rules

### Sidebar

- navegacion agrupada
- active state con color de marca
- branding arriba
- cuenta y settings abajo

### Topbar

- titulo de pagina
- search compacta
- workspace switcher
- CTA de accion rapida
- notificaciones

## Interaction patterns

- hover suave en cards y filas
- transiciones rapidas
- animaciones de entrada discretas
- forms simples, sin complejidad artificial

## Data display rules

- metricas arriba
- visualizacion principal al centro
- resumenes y tablas en la parte inferior
- estados con badges o texto de color

## Expansion guidance

- extender sobre el layout actual, no crear shells nuevos
- si un modulo necesita mas profundidad, empezar por tabs o bloques internos antes de crear nuevas rutas
- mantener una sola direccion visual basada en la base oficial actual
